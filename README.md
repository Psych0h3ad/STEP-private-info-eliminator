## 日本語

**STEP Sanitizer** は、STEP (ISO 10303-21) ファイルに埋め込まれた作成者・組織・CADソフト名・作業日時・保存パス・フィーチャー名などの識別情報を除去します。形状データ(ジオメトリ)は1バイトも変更しません。

**Web版**: `index.html` をブラウザで開くだけ(または GitHub Pages の URL)。**ファイルはどこにも送信されません。**
**CLI版**: `step_sanitize.py`(Python 3.8+、依存パッケージなし)

### なぜ安全と言えるか

- 静的HTML 1ファイルで完結し、サーバー側の処理が存在しない = 送信しようがない設計
- 外部リソース(CDN・Webフォント・解析タグ)を一切読み込まない → ページを開いた後は機内モードでも動作
- 開発者ツールの Network タブで通信ゼロを誰でも検証可能
- Web版とCLI版は同一ロジックで、出力はバイト単位で一致(テスト済み)

### 検出・除去できるもの

| 場所 | 内容 |
|---|---|
| HEADER `FILE_NAME` | 作成者・組織・出力ソフト(preprocessor / originating_system)・承認者・タイムスタンプ・**保存時のフルパス**(`C:\Users\本名\案件名\...`) |
| HEADER `FILE_DESCRIPTION` | 説明欄(ソフト名が入りがち) |
| DATA | `PERSON` / `ORGANIZATION` / 住所エンティティの文字列(AP203系) |
| DATA | `CALENDAR_DATE` / `LOCAL_TIME`(作業タイムライン)→ 1970-01-01 に置換 |
| DATA | `\X2\` エスケープの非ASCII文字列(**フィーチャー名**など。作成環境の言語が推定できる) |
| DATA | `PRODUCT` 名(部品名・案件名)の付け替え(任意) |
| 全体 | `/* Generated by ... */` 系コメント |

### 使い方

Web版: ページを開き、`.step` / `.stp` をドロップ → 検出結果を確認 → クリーンなコピーをダウンロード。まずは `demo.step`(架空データ)で試せます。

CLI版:

```bash
python3 step_sanitize.py input.step --scan     # 何が入っているか確認
python3 step_sanitize.py input.step            # → input_clean.step
python3 step_sanitize.py input.step --name part.step --product part-A
```

### セルフホスト(GitHub Pages)

Settings → Pages → **Deploy from a branch** → `main` / `(root)` を選択して Save。静的ファイルのみなのでビルド不要です。

### 制限

- メタデータを消しても、エンティティの記述スタイル(書き癖)から出力元CADがある程度推定できる場合があります。痕跡を完全に断つには別のCADカーネル(FreeCAD / build123d 等)で再エクスポートしてください(形状の再検証を推奨)
- Web版の目安は 100MB 程度まで(それ以上はCLI版を推奨)
- 対応: テキスト形式の Part 21(`.step` / `.stp` / `.p21`)。AP203 / AP214 / AP242 などスキーマは問いません。ZIP圧縮(`.stpZ`)・gzip は CLI版が自動解凍(Web版は解凍してから)。バイナリ(Part 26)・XML(Part 28)は非対応

## English

**STEP Sanitizer** removes identifying metadata embedded in STEP (ISO 10303-21) files — author, organization, CAD software, timestamps, save paths, feature names — without changing a single byte of geometry.

**Web version**: just open `index.html` (or the GitHub Pages URL). **Your file never leaves the browser.**
**CLI version**: `step_sanitize.py` (Python 3.8+, zero dependencies)

### Why you can trust it

- A single static HTML file with no server-side code — uploading is structurally impossible
- No external resources (CDNs, web fonts, analytics) — works in airplane mode once loaded
- Verify zero network traffic yourself in DevTools → Network
- Web and CLI share the same logic; outputs are byte-identical (tested)

### What it detects & removes

| Where | What |
|---|---|
| HEADER `FILE_NAME` | author, organization, exporting software (preprocessor / originating_system), approver, timestamp, and the **full save path** (`C:\Users\realname\ClientX\...`) |
| HEADER `FILE_DESCRIPTION` | description field (often contains software names) |
| DATA | strings in `PERSON` / `ORGANIZATION` / address entities (AP203) |
| DATA | `CALENDAR_DATE` / `LOCAL_TIME` (your work timeline) → replaced with 1970-01-01 |
| DATA | non-ASCII strings in `\X2\` escapes (**feature names** — they reveal the language of your CAD environment) |
| DATA | `PRODUCT` name (part/project name) — optional rename |
| Global | `/* Generated by ... */` comments |

### Usage

Web: open the page, drop a `.step` / `.stp` file, review the report, download the cleaned copy. Try it first with `demo.step` (fictional data).

CLI:

```bash
python3 step_sanitize.py input.step --scan     # inspect only
python3 step_sanitize.py input.step            # → input_clean.step
python3 step_sanitize.py input.step --name part.step --product part-A
```

### Self-hosting (GitHub Pages)

Settings → Pages → **Deploy from a branch** → `main` / `(root)`, then Save. Static files only, no build step.

### Limitations

- Even with metadata removed, the exporting CAD can sometimes be inferred from entity writing style. For complete trace removal, re-export through another CAD kernel (FreeCAD / build123d) and re-verify the geometry
- Web version: up to ~100 MB recommended (use the CLI for larger files)
- Supported: text Part 21 (`.step` / `.stp` / `.p21`), any schema (AP203 / AP214 / AP242 …). ZIP-compressed (`.stpZ`) and gzip are auto-extracted by the CLI (unzip first for the web version). Binary (Part 26) and XML (Part 28) are not supported

## 中文

**STEP Sanitizer** 用于移除 STEP (ISO 10303-21) 文件中嵌入的身份识别元数据——作者、组织、CAD 软件、时间戳、保存路径、特征名等——几何数据一个字节都不会改动。

**网页版**:直接打开 `index.html`(或 GitHub Pages 链接)。**文件不会离开你的浏览器。**
**命令行版**:`step_sanitize.py`(Python 3.8+,零依赖)

### 为什么可信

- 仅一个静态 HTML 文件,不存在服务器端代码——从结构上就无法上传
- 不加载任何外部资源(CDN、网页字体、统计脚本),页面打开后开飞行模式也能用
- 可自行在开发者工具 → Network 面板验证零网络请求
- 网页版与命令行版共用同一套逻辑,输出逐字节一致(已测试)

### 检测与移除的内容

| 位置 | 内容 |
|---|---|
| HEADER `FILE_NAME` | 作者、组织、导出软件(preprocessor / originating_system)、审批人、时间戳,以及**完整保存路径**(`C:\Users\真名\客户项目\...`) |
| HEADER `FILE_DESCRIPTION` | 描述字段(常含软件名) |
| DATA | `PERSON` / `ORGANIZATION` / 地址实体中的字符串(AP203) |
| DATA | `CALENDAR_DATE` / `LOCAL_TIME`(工作时间线)→ 替换为 1970-01-01 |
| DATA | `\X2\` 转义的非 ASCII 字符串(**特征名**——会暴露建模环境的语言) |
| DATA | `PRODUCT` 名(零件/项目名)可选改名 |
| 全局 | `/* Generated by ... */` 注释 |

### 使用方法

网页版:打开页面,拖入 `.step` / `.stp` 文件,查看检测报告后下载清理后的副本。可先用 `demo.step`(虚构数据)体验。

命令行:

```bash
python3 step_sanitize.py input.step --scan     # 仅检测
python3 step_sanitize.py input.step            # → input_clean.step
python3 step_sanitize.py input.step --name part.step --product part-A
```

### 自行部署(GitHub Pages)

Settings → Pages → **Deploy from a branch** → 选择 `main` / `(root)` 后 Save。纯静态文件,无需构建。

### 限制

- 即使清除了元数据,仍可能从实体的书写风格推断出导出软件;如需彻底消除痕迹,请用其他 CAD 内核(FreeCAD / build123d 等)重新导出,并复核几何
- 网页版建议 100 MB 以内(更大文件请用命令行版)
- 支持:文本 Part 21(`.step` / `.stp` / `.p21`),不限 schema(AP203 / AP214 / AP242 等)。ZIP 压缩(`.stpZ`)与 gzip 由命令行版自动解压(网页版请先解压)。不支持二进制(Part 26)与 XML(Part 28)
